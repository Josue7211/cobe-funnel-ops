# 13-02: Show Branch Decisions

## Goal
Make orchestration logic visible inside each run.

## Depends on
- 13-01

## Requirements
- ORCH-01

## Success Criteria
1. Branches, filters, and actions are visible.
2. The path taken by a run is obvious.

## Plan
- Break the run into a four-step trace: trigger, filter, action, downstream.
- Show the current connector branch decision next to the latest run context.
- Keep the sequence compact and traceable inside the existing inspector card.
