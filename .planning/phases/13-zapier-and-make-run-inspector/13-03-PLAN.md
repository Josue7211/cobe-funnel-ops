# 13-03: Add Retry and Replay Actions

## Goal
Let operators rerun failed orchestration work.

## Depends on
- 13-01 and 13-02

## Requirements
- ORCH-02

## Success Criteria
1. Failed runs can be replayed or retried.
2. The UI makes the action result visible.

## Plan
- Add rerun and retry controls inside the connector inspector.
- Wire actions to the selected connector, latest run, and active lead context.
- Show the latest action outcome and failed-run trail clearly.
