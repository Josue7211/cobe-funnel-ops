# 12-03: Link Cross-System Surfaces

## Goal
Make every lifecycle node link to the underlying surface (DM workbench, booking, payment, onboarding, alerts) so the operator can hop between systems without losing context.

## Depends on
- 12-01 and 12-02

## Requirements
- LIFE-01, LIFE-02

## Success Criteria
1. Lifecycle nodes include links/buttons that focus the relevant panel (DM, queue, reports, onboarding proof, alerts) when clicked.
2. Each link is annotated with the reason the event matters (e.g., “queued no-show recovery”).
3. The automation rail and queue filters honor the same link so clicking a node syncs the rest of the console.

## Plan
- Add helper metadata to lifecycle nodes (target rail, highlight action) and use it to drive the button that switches `railTab` or `workbenchTab`.
- Wire the “Focus lead” buttons inside the inbox summary and lifecycle strip so the same explanation text appears in both places.
- Keep the alert/connector rail up to date by refreshing the integration events when connectors change status.
