# 12-04: Prioritize Lifecycle Clarity

## Goal
Strip away filler chrome from the active workflow pane so the lifecycle story reads first and actions stay accessible.

## Depends on
- 12-01 to 12-03

## Requirements
- LIFE-01, LIFE-02

## Success Criteria
1. The workflow header and reusable lifecycle strip occupy the first viewable space; the queue/timeline sections only appear once the lead is clear.
2. Workflow actions are grouped around the lifecycle nodes rather than hidden in unrelated rails.
3. The rail layout stays dense but responsive at desktop widths and collapses gracefully on narrow screens.

## Plan
- Move the “Active workflow” header and lifecycle strip above the stage layout; keep secondary rails (tools, metrics) collapsed until a lead is selected.
- Use the new `activeLeadLifecycleNodes` to annotate the “Active workflow” callouts and automation summaries with the same status text.
- Tighten padding/gap values for the workflow panels so the timeline/queue stack remains readable without extra scrolling.
