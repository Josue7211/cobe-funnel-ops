# 12-01: Build Lead Lifecycle Graph

## Goal
Surface one lead's full integration journey with a compact, ordered graph that spans intake, routing, revenue, and onboarding.

## Depends on
- Phase 11

## Requirements
- LIFE-01

## Success Criteria
1. The lifecycle strip highlights the most recent lead-linked integration events in chronological order.
2. Each node clearly labels the system (DM, booking, payment, onboarding, alert) and why it matters.
3. Nodes feed the active workflow pane so operators can focus on the timeline before supporting chrome.

## Plan
- Treat `integrationEvents` as the canonical event stream; sort the newest six for the active lead and classify them into phases (intake, booking, payment, onboarding, alert, routing).
- Build the UI strip inside the workflow header: show mini-label, status badge, timestamp, and a one-line effect summary for each node.
- Allow clicking a lifecycle node to set the active lead (reuse the existing focus button) and keep the graph visually tied to the events that feed the queue/timeline.
